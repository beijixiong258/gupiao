# A 股 T+3 量化研究员：项目设计与实现

> 本文面向 AI 编程助手和开发者，记录产品边界、数据流、量化方法、模块职责、完成状态与后续计划。普通用户的安装、配置和命令说明见 [README.md](./README.md)。

## 1. 项目定位

这是一个只研究中国大陆 A 股的本地命令行智能体。用户用自然语言连续提问，DeepSeek 或 OpenAI 负责理解问题、识别前文指代和解释结果，确定性 Python 工具负责拉取数据、计算指标、训练模型和提供证据。

程序只处理两类任务：

1. **单股研究**：询问某只 A 股怎么样，返回基本资料、估值、财务指标、技术指标、风险和数据来源。
2. **板块选股与短线预测**：从指定行业或概念板块中筛选股票，分别预测未来 T+1、T+2、T+3 个交易日的累计收盘收益，并根据样本外验证决定是否允许给出研究候选。

### 永久安全边界

本项目被代码硬限制为 `research_only`：

- 不连接券商；
- 不读取或保存证券账户、交易密码；
- 不提交、修改或撤销委托；
- 不控制交易软件；
- 不做自动交易；
- 不预留“以后打开自动交易”的配置开关。

程序输出的候选、持有天数和价格预测都只供人工研究。最终是否交易、何时交易、交易多少，必须由用户在项目之外人工决定。

## 2. 总体流程

```text
用户自然语言
    |
    v
CLI 会话层（载入并保存上下文）
    |
    v
DeepSeek / OpenAI 判断任务类型
    |
    +-- 单只股票问题 ------> gupiao_fenxi
    |
    +-- 指定板块选股问题 --> bankuai_xuangu
                                |
                                v
                    T+1 / T+2 / T+3 样本外验证
                                |
                +---------------+---------------+
                |                               |
             通过门槛                         未通过门槛
                |                               |
          输出研究候选                    明确输出“不推荐”
```

智能体默认只暴露两个业务工具。它不能调用 Shell、后台任务、文件编辑器或任何交易接口来回答股票问题。

### 2.1 连续会话

终端会话不是简单地重复执行单轮命令：

1. `DuihuaCunchu` 从 `%USERPROFILE%\.gupiaoyanjiu\duihua\` 载入最近会话或指定会话。
2. 每轮把此前的用户消息、助手回答、工具调用和工具结果交给 `ContextBuilder`。
3. `ContextBuilder` 丢弃历史中的 `system` 消息，只接受 `user`、`assistant`、`tool`，防止会话文件覆盖系统安全边界。
4. `AgentLoop` 每轮创建独立运行目录，执行 ReAct 工具循环，并返回去掉系统提示词后的完整历史。
5. CLI 仅在该轮成功后更新会话，采用 UTF-8 JSON 临时文件原子替换；中断或写入失败不会破坏上一轮历史。
6. 上下文过长时沿用工具结果裁剪、旧文本折叠和 LLM 摘要机制，压缩后的历史继续写回会话。

因此后续问题可以引用“它”“刚才那只”“第二只”等前文对象。会话存储不包含 API Key、Tushare Token 或证券账户信息。

## 4. 数据来源与权衡

### 4.1 `source=auto` 的原则

系统不是简单地“有 Tushare 就什么都用 Tushare”，而是按数据类型选择：

| 数据 | 首选 | 降级顺序 |
|---|---|---|
| 行业成分 | Tushare `stock_basic.industry` 本地缓存 | AKShare 新浪板块 -> AKShare 东方财富 |
| 概念成分 | AKShare 新浪板块 | AKShare 东方财富 |
| 个股日线 | Tushare `daily` | AKShare 新浪前复权 -> AKShare 东方财富前复权 |
| 复权因子 | Tushare `adj_factor` | AKShare 前复权日线 |
| 基本资料 | Tushare `stock_basic` 缓存 | AKShare 股票代码表/个股资料 |
| 估值 | Tushare `daily_basic` | AKShare 实时估值 |
| 财务指标 | Tushare `fina_indicator` | AKShare 财务分析指标 |

每次结果都会返回 `source`、`adjustment`、`warnings` 和 `errors`。付费接口没有权限、被限频或返回空值时，程序会说明原因后降级，不会把免费数据伪装成 Tushare 数据。

### 4.2 120 积分环境的实际表现

2026-07-15 在当前电脑和 Token 上实测：

- Tushare `daily` 可用；
- Tushare `daily_basic` 可返回估值；
- `stock_basic` 和 `adj_factor` 频率较低，必须缓存或降级；
- `fina_indicator` 无权限；
- 财务指标可降级到 AKShare；
- Clash 全局模式下，东方财富部分域名会出现代理 443/断连；
- `lianghua_peizhi.json` 默认让 AKShare 国内接口临时绕过代理，调用结束后恢复代理，因此不会影响 DeepSeek/OpenAI。

权限和接口规则可能由数据服务商调整，应以每次工具实际返回为准。

## 5. 单股分析

调用入口：`gupiao_fenxi`

输入可以是 `600519.SH`、`600519` 或中文股票名。代码和名称会先标准化，并限制在沪深北普通 A 股范围内。

### 5.1 技术指标

系统从前复权日线计算：

- 1、3、5、10、20 日收益；
- MA5、MA10、MA20、MA60 及均线位置；
- RSI(14)；
- MACD DIF、DEA、柱值；
- ATR(14) 占价格比例；
- 20 日年化波动率；
- 距 20 日高点回撤、20 日区间位置；
- 5 日与 20 日成交量比；
- 20 日支撑位和阻力位。

技术评分由固定规则计算，LLM 只能解释评分证据，不能修改分数。

### 5.2 基本面

系统尝试返回：

- 名称、地区、行业、市场、上市日期；
- PE、PE-TTM、PB、总市值、流通市值、换手率；
- ROE、毛利率、净利率、资产负债率；
- 营收同比、净利润同比、EPS、经营现金流质量。

接口没有权限时，相应字段为空并列出错误。空字段不参与评分，LLM 不得补写。

## 6. 板块选股与预测

调用入口：`bankuai_xuangu`

### 6.1 板块与初筛

1. 根据名称匹配行业或概念板块，并返回最终识别的板块名、类型和相似度。
2. 删除 ST、退市标记和上市初期特殊简称。
3. 应用价格与成交额下限。
4. 删除最新交易日接近涨停、次日难以成交的股票。
5. 按流动性限制历史行情拉取数量，默认最多研究 20 只，避免接口失控和长时间无响应。

### 6.2 模型特征

每个交易日使用 19 个只依赖当日及过去数据的特征：

- 1/3/5/10/20 日动量；
- 价格相对 MA5/10/20/60 的距离；
- MA5 相对 MA20 的趋势；
- RSI、MACD；
- ATR、20 日波动率；
- 20 日回撤和区间位置；
- 量能比和单日振幅。

目标分别是从当日收盘到未来第 1、2、3 个交易日收盘的累计收益。三个周期分别训练 `HistGradientBoostingRegressor`，不会用 T+3 标签代替 T+1。

### 6.3 防止“回测看起来很好”

训练和验证按交易日期排序，不随机打乱。切分点之前的训练样本如果其目标日期跨过切分点，也会被删除，避免未来收益泄漏到训练集。

每个周期返回：

- 训练和验证样本数；
- MAE；
- 中位数基线 MAE；
- 相对基线提升；
- 涨跌方向准确率；
- 按日横截面 Rank IC；
- 预测裁剪范围；
- `high` / `medium` / `low` 质量标签。

只有满足以下所有条件才进入 `recommendations`：

1. 至少一个周期通过样本外验证；
2. 建议卖出周期本身通过验证；
3. 加权成本后收益为正；
4. 建议周期成本后收益为正；
5. 扣除 ATR 和波动率惩罚后的选择分数为正。

否则返回成功状态和 `no_trade_reason`，推荐数为 0。`watchlist` 仅用于查看模型原始排序，不代表推荐。

### 6.4 当前局限

- 使用当前板块成分股回看历史，仍存在幸存者偏差；
- 一次留出验证不等于跨多个牛熊周期稳定；
- 日线收盘预测不能保证次日开盘可按该价格成交；
- 小样本、板块快速轮动、政策和突发事件会使模型失效；
- “预测到一个数字”不代表具有可交易优势，必须看样本外指标。

## 7. A 股规则与成本

- 当日买入最早下一个交易日卖出，即 T+1；
- 主板、ST、创业板/科创板、北交所使用不同涨跌幅限制；
- 普通竞价买入通常按 100 股整数倍申报；
- 佣金通常买卖双向收取，并可能有每笔最低收费；
- 印花税只在卖出时收取；
- 滑点表示模型价格与实际成交价格之间的偏差。

印花税必须计入，因为 T+1 到 T+3 的预期收益本来就小。忽略卖出印花税、最低佣金和滑点，会把本来不可交易的微弱毛收益误判成盈利。

成本参数在根目录 [jiaoyi_chengben.json](./jiaoyi_chengben.json)，量化参数在 [lianghua_peizhi.json](./lianghua_peizhi.json)。税费和券商费率变化时，应修改配置后重新研究。

## 8. Provider 与身份数据

支持的 Provider 只有：

- `deepseek`：DeepSeek API Key；
- `openai`：OpenAI API Key；
- `openai_codex`：ChatGPT Pro / Codex OAuth。

Provider 和 Tushare 配置位于 `agent/.env`。OAuth 凭据位于 `%USERPROFILE%\.gupiaoyanjiu\openai_codex_auth.json`，会话位于 `%USERPROFILE%\.gupiaoyanjiu\duihua\`。这些运行数据都不应写入源码文件。

OpenRouter 及其他通用 Provider 路由不在产品范围内。

## 9. MCP 对外接口

MCP 客户端负责理解自然语言，本服务只负责取数和计算。支持 stdio 以及本机 HTTP `/mcp`。

MCP 永远只列出两个工具：

- `gupiao_fenxi`：单股基本面和技术面研究；
- `bankuai_xuangu`：指定板块选股及 T+1/T+2/T+3 预测。

两个工具均为只读、非破坏性。MCP 不提供 Shell、文件编辑、券商连接、账户读取、订单提交或自动交易能力。

## 10. 外置配置

`lianghua_peizhi.json` 控制：

- 历史天数、最多研究股票数、最少有效日线；
- AKShare 是否临时绕过代理；
- 最低价格、最高价格、最低成交额；
- 模型迭代、树深、叶节点、验证比例、最低样本数；
- T+1/T+2/T+3 权重；
- 成本场景；
- 强制 `research_only` 和禁止订单提交。

配置中的预测周期必须严格为 `[1, 2, 3]`。若把自动交易开关改成 `true`，程序会直接判定配置无效，而不是启用交易。
