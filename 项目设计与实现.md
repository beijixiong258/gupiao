# A 股 T+3 量化研究员：项目设计与实现

> 本文面向 AI 编程助手和开发者，记录产品边界、数据流、量化方法、模块职责、完成状态与后续计划。普通用户的安装、配置和命令说明见 [README.md](./README.md)。

## 1. 项目定位

这是一个只研究中国大陆 A 股的本地命令行智能体。用户用自然语言连续提问，DeepSeek 或 OpenAI 负责理解问题、识别前文指代和解释结果，确定性 Python 工具负责拉取数据、计算指标、训练模型和提供证据。

量化工具只处理两类分析任务：

1. **单股研究与短线预测**：先完成行情时点、可交易性、基本面、估值、技术面、同行和风险诊断；只有用户确实询问未来走势、盈利空间、买卖或具体 T+1～T+3 数值时，才用同一分析编号执行第二阶段预测。未验证或时点失效的内部点预测不向用户发布。
2. **板块筛选与短线预测**：从指定行业或概念板块中筛选股票，在 T 日完整收盘后形成分析截面，以下一交易日开盘为持有期情景的测算基准，预测 T+1、T+2、T+3 三个周期的累计收益，并仅把通过样本外验证的对象列为研究候选。

单股和板块都支持 T+1/T+2/T+3，但用途不同：单股链路只评估用户指定的一只股票和期限，板块工具负责在明确板块内排序。用户自然语言中的“能不能买”解释为扣除广义交易成本后是否仍有上涨空间；“能不能卖”先分析剩余上涨空间，再结合用户买入价和持仓规模计算净盈亏。这里的买卖只是分析问题，不是软件参与交易决策。技术分和基本面分只作解释证据，不能冒充上涨概率或期限预测。

### 永久安全边界

本项目被代码硬限制为 `research_only`：

- 不连接券商；
- 不读取或保存证券账户、交易密码；
- 不提交、修改或撤销委托；
- 不控制交易软件；
- 不做自动交易；
- 不预留“以后打开自动交易”的配置开关。

程序输出的候选、持有天数和价格预测都只供人工研究。最终是否交易、何时交易、交易多少，必须由用户在项目之外人工决定。

## 2. 总体流程

```text
用户自然语言
    |
    v
CLI 会话层（载入并保存上下文）
    |
    v
DeepSeek / OpenAI 按完整语义选择交互链路
    |
    +-- 需要量化分析 ------> 判断单股或板块工作流
    |                          |
    |                          +-- 单股：gupiao_fenxi 全面诊断
    |                          |             |
    |                          |             +-- 需要具体预测数值：gupiao_yuce
    |                          |
    |                          +-- 板块：bankuai_xuangu（每批最多 8，只按稳定序列顺延）
    |
    +-- 不需要量化工具 ----> 直接聊天，不调用工具
                               |
                               +-- 主业相关：简洁解释
                               +-- 主业无关：一句话提示回到 A 股分析与预测
```

智能体默认只暴露三个业务工具：`gupiao_fenxi`、`gupiao_yuce`、`bankuai_xuangu`。它不能调用 Shell、后台任务、文件编辑器或任何交易接口来回答股票问题。

### 2.1 连续会话

终端会话不是简单地重复执行单轮命令：

1. `DuihuaCunchu` 从 `%USERPROFILE%\.gupiaoyanjiu\duihua\` 载入最近会话或指定会话。
2. 每轮把此前的用户消息、助手回答、工具调用和工具结果交给 `ContextBuilder`。
3. `ContextBuilder` 丢弃历史中的 `system` 消息，只接受 `user`、`assistant`、`tool`，防止会话文件覆盖系统安全边界。
4. `AgentLoop` 每轮创建独立运行目录，执行 ReAct 工具循环，并返回去掉系统提示词后的完整历史。
5. CLI 仅在该轮成功后更新会话，采用 UTF-8 JSON 临时文件原子替换；中断或写入失败不会破坏上一轮历史。
6. 上下文过长时只压缩本轮供模型继续推理的工作副本；完整工具结果和完整会话历史仍单独保存。

因此后续问题可以引用“它”“刚才那只”“第二只”等前文对象。程序不会主动从配置文件读取密钥并写入会话或运行目录，但用户消息、助手回答和工具结果会原样保存；用户不得在对话中粘贴 API Key、Tushare Token、交易密码或其他敏感信息。

第一阶段单股结果带有独立契约版本和 `analysis_id`。载入旧会话时，缺少当前契约版本、分析编号或已完成诊断阶段标记的历史单股结果会保留原有工具调用结构，但内容被替换成“过期结果”占位；由该旧结果生成的历史文字结论也会被失效占位替换。同一股票、同一完整收盘快照和同一进程内的兼容分析编号可用于解释性追问或第二阶段数值计算；进程重启、股票变化、来源变化、行情更新或时点失效后必须重新诊断。是否进入量化工具链路由智能体根据完整语义、会话对象、数据时效、持有期限和已有结果判断，不使用关键词或正则表达式硬路由；无需工具时直接聊天，主业无关问题仅作一句话提示。

## 4. 数据来源与权衡

### 4.1 `source=auto` 的原则

系统不是简单地“有 Tushare 就什么都用 Tushare”，而是按数据类型选择：

| 数据 | 首选 | 降级顺序 |
|---|---|---|
| 行业成分 | Tushare `stock_basic.industry` 24 小时本地缓存 | AKShare 新浪板块 -> AKShare 东方财富 |
| 概念成分 | AKShare 新浪板块 | AKShare 东方财富 |
| 个股日线 | 12 小时单股缓存 -> 覆盖完整的全市场日频仓库 -> Tushare `daily` | AKShare 新浪前复权 -> AKShare 东方财富前复权 |
| 复权因子 | 全市场日频仓库或 Tushare `adj_factor` | AKShare 前复权日线 |
| 基本资料 | Tushare `stock_basic` 24 小时缓存 | AKShare 股票代码表/个股资料 |
| 估值 | Tushare `daily_basic` | AKShare 实时估值 |
| 财务指标 | Tushare `fina_indicator` | AKShare 财务分析指标 |
| 当前执行快照 | AKShare 东方财富全市场快照 | 无；不可用时退回最近完整日线并明确标记 |
| 单股同行横截面 | Tushare `stock_basic` + `daily` + 可用时的 `daily_basic` | AKShare 当前全市场快照 + 本地股票代码表 |

成功结果会提供实际数据来源、复权口径和降级/错误信息，但单股与板块工作流的字段层级不同。付费接口没有权限、被限频或返回空值时，程序会说明原因后降级，不会把免费数据伪装成 Tushare 数据。

单股命令显式选择 `tushare` 或 `akshare` 时，会固定股票名称解析和日线来源；`auto` 才按 Tushare 优先、AKShare 降级。板块命令的 `source` 只控制成分股日线。基本面和板块成分有各自的数据源策略，因此显式选择 `akshare` 也不表示整份结果只访问 AKShare。

板块模型不混入未复权日线。`auto` 模式下某一只股票的 Tushare 复权失败时，只为该股票尝试 AKShare 前复权，不会因为一只股票失败就改变后续所有股票的数据源；仍无法取得复权行情时排除该股票并记录原因。

### 4.2 120 积分环境的实际表现

2026-07-15 在当前电脑和 Token 上实测：

- Tushare `daily` 可用；
- Tushare `daily_basic` 可返回估值；
- `stock_basic` 和 `adj_factor` 频率较低，必须缓存或降级；
- `fina_indicator` 无权限；
- 财务指标可降级到 AKShare；
- Clash 全局模式下，东方财富部分域名会出现代理 443/断连；
- `lianghua_peizhi.json` 默认让 AKShare 国内接口临时绕过代理，调用结束后恢复代理，因此不会影响 DeepSeek/OpenAI。

权限和接口规则可能由数据服务商调整，应以每次工具实际返回为准。

### 4.3 全市场日频仓库

`riping_cangku.py` 使用项目内的 SQLite 文件 `agent/cache/a_share_daily_warehouse.sqlite3` 建立可恢复的日频数据底座。它严格按市场交易日调用并保存：

- Tushare `daily`：全市场未复权开高低收、成交量和成交额；
- Tushare `adj_factor`：同交易日复权因子，读取单股历史时相对区间末日因子计算前复权价格；
- Tushare `daily_basic`：同交易日换手、估值、股本和市值；
- Tushare `trade_cal`：开闭市日历；
- Tushare `stock_basic`：上市、退市、暂停上市股票资料，按实际抓取日期保留快照。

`gpyj warehouse sync --start YYYY-MM-DD` 默认从最近缺口向前处理 20 个交易日。三个日频接口分别记录完成、失败和行数；一次只有 `daily_basic` 失败时，下一次不会重复请求已经完成的 `daily` 与 `adj_factor`。`--oldest-first` 可改为从旧到新，`--max-sessions 0` 可处理全部缺口，`--force` 用于明确重抓已完成日期。`gpyj warehouse status` 返回覆盖日期、完整交易日、各表行数、文件大小和股票资料快照日期；`gpyj settings` 同时给出摘要。

低积分账号可能只有 `daily_basic` 每分钟 1 次、`adj_factor` 每分钟 1 次且每天 5 次，按交易日全量回补不可执行。`--price-only` 模式只持久化全市场 `daily`；读取某只股票时才在约千行时间序列上递推相对复权因子：`factor_t / factor_(t-1) = 上一实际收盘价 / 当日pre_close`，不为全市场永久保存一张重复派生表。该比例保证除权日前后的模型收益与数据源 `pct_chg` 口径连续，不需要知道复权因子的绝对基准。低积分账号默认单线程；可选并发受全局每分钟最多30次启动限制，SQLite始终单线程提交。历史 `daily_basic` 保持缺失，不做插值或伪造。

仓库不是模型的强制前置条件。读取优先级为 12 小时单股文件缓存、覆盖完整的 SQLite 仓库、远程 Tushare/AKShare 降级链。请求区间至少包含 60 个交易日，且 `daily` 与同一口径复权链的完整交易日比例达到 90% 时，才把仓库视为可扩容面板：单股上限从 20 提高到 60，同业优先数从 16 提高到 45；板块上限从 24 提高到 80。`daily_basic` 覆盖率独立报告，缺失的估值/规模/换手特征仍由训练窗覆盖率与稳定性筛选排除。价格覆盖不足、数据库不存在或读取失败时，模型保持原规模，不会用残缺仓库替代远程数据。

这里的 PIT 边界必须准确表述：日线、复权因子、可取得的 `daily_basic` 和日历可以按历史交易日保存；股票资料只能从仓库建立之日起形成真实快照。系统不会把当前行业标签倒填到过去，也无法从现有接口倒推出仓库建立前的历史板块成分。低额度模式解决前复权价格和更大日频横截面问题，但不解决历史估值缺口；两种模式都不能彻底消除当前成分回看历史造成的幸存者偏差。

## 5. 单股分析

第一阶段入口：`gupiao_fenxi`；第二阶段数值入口：`gupiao_yuce`。

输入可以是 `600519.SH`、`600519` 或中文股票名。代码和名称会先标准化，并限制在沪深北普通 A 股范围内；名称包含匹配出现多个候选时返回歧义错误，不会静默选择第一只股票。只有已经确认收盘的日线参与计算，行情超过 7 个工作日仍未更新时停止输出“当前分析”。

### 5.1 技术指标

系统从前复权日线计算：

- 1、3、5、10、20 日收益；
- MA5、MA10、MA20、MA60 及均线位置；
- RSI(14)；
- MACD DIF、DEA、柱值；
- ATR(14) 占价格比例；
- 20 日年化波动率；
- 距 20 日高点回撤、20 日区间位置；
- 5 日与 20 日成交量比；
- 20 日最低价和最高价参考区间。

技术评分由固定规则计算，LLM 只能解释评分证据，不能修改分数。历史不足 60 日时 MA60、MACD 等未形成指标保持为空并列出提示，不会自动当成负面信号；横盘 RSI 按中性值处理。

### 5.2 基本面

系统尝试返回：

- 名称、地区、行业、市场、上市日期；
- PE、PE-TTM、PB、总市值、流通市值、换手率；
- ROE、毛利率、净利率、资产负债率；
- 营收同比、净利润同比、EPS、经营现金流质量。

Tushare 财务指标只使用公告日和报告期都不晚于分析日的记录；无公告日的 AKShare 财务数据只用于当前时点降级。接口没有权限时，相应字段为空并列出错误，空字段不参与评分，LLM 不得补写。

基本面 `score_0_100` 是启发式检查分，不是上涨概率，也没有完成全行业同行排名。金融行业资产负债率口径特殊，该字段只展示、不按普通工业公司阈值加减分。

### 5.3 “现在”和“今天”的时间口径

程序先记录上海时区的生成时间并区分开盘前、集合竞价、盘中、午休、刚收盘和盘后。模型永远只使用数据源已经确认完成的日线；盘中 AKShare 快照只用于检查当前价格、成交额和是否接近涨停，不进入模型训练，也不会被称为完整收盘价。快照接口不提供逐行交易日期时，结果会明确标为“尽力而为的当前快照”。如果最近完整收盘信号对应的下一交易日开盘已经过去，当前盘中价格不再符合持有期情景的入口口径，结果会标记“原测算入口已失效，需要最新完整收盘后重算”。

可交易性闸门检查：

- 行情是否严重滞后或疑似停牌；
- ST、退市风险和 N/C 新股特殊阶段；
- 上市历史是否达到最少 180 个自然日；
- 最近可用成交额是否达到单股配置门槛；
- 当前是否一字或近似一字涨停、是否接近涨停；
- 目标资金是否满足主板/创业板、科创板或北交所最低买入数量。

约束未通过时仍可展示基本面和技术面，但必须把对应限制列为负面证据，LLM 不得隐去或改写。

### 5.4 单股同行模型

用户指定的持有期限按完整语义映射到对应模型，例如“持有两个交易日”对应 T+2；这不是用正则表达式强制调用。程序优先从当前同行业中选择高流动性股票，再用少量全市场高流动性股票补足；目标股票无论流动性排名如何都会保留。仓库覆盖不足时最多 20 只；请求历史区间的日频仓库覆盖达标后最多 60 只，其中同业优先数由 16 提高到 45。两种模式都至少需要 8 只有效同行历史，每只使用前复权日线。

单股模型除复用 19 个基础技术特征外，还使用：

- 开盘缺口、日内收益、收盘在当日区间的位置；
- 成交额对数和 5/20 日成交额比；
- 同行 1/5/20 日平均收益以及目标相对同行的超额收益；
- 同行多头宽度、5 日正收益宽度和收益离散度；
- 动量、MA20 偏离、量能、波动率和成交额的横截面百分位。
- 上证、沪深300和中证1000的1/5/20日收益及20日波动率；
- 当前同行行业代理与市场参考组的日频收益、宽度、分散度和相对强弱；
- 可取得时严格同日匹配的换手率、盈利收益率、账面市值比、流通市值对数、横截面排名及流通市值中性残差。

所有横截面特征只使用同一信号日当时的数据。模型由 `HistGradientBoostingRegressor` 和“中位数填补 + 稳健缩放 + L2正则化Ridge”组成小型集成：树模型负责非线性和特征交互，Ridge提供低方差线性约束，避免单一树模型在小面板上过度追随局部模式。

进入模型前，先在当前训练窗内按交易日检查因子覆盖率，并比较三个连续时间片的Rank IC方向；覆盖不足或极不稳定的因子不会进入本折，数量不足时才按训练窗证据最强的因子保底。入选特征再按当前训练窗自身的1%和99%分位数去极值；验证窗、最终保留窗和最新预测不能参与筛选或边界计算。Ridge与方向Logistic随后做10%～90%分位范围的稳健缩放，树模型不做无意义的尺度标准化。

集成不是固定平均。单股滚动验证的第一折使用保守默认权重，后续每折只能用更早折已经产生的样本外预测，在有限权重网格中按 MAE 选择树模型权重；最终生产权重使用全部滚动样本外预测。当前折的真实收益不能参与当前折权重选择，最终保留窗口仍是未被自身答案调参的验证窗口。结果同时记录两类模型的权重、各自样本外 MAE 和预测分歧。

### 5.5 滚动验证、成本与证据评估

每个 T+1/T+2/T+3 模型使用三段扩展窗口滚动验证，最后一段单独标为最终保留测试窗口且必须通过。每一折训练集只允许包含信号日和目标退出日都早于验证起点的样本，避免跨界标签泄漏；全部样本外指标完成后，再使用截至当前已经有结果的全部标签重训生产模型。

模型门槛同时检查方向准确率、相对中位数基线提升、每日 Rank IC、Top-N 扣成本收益、相对同行超额，以及三折中至少两折具有正向能力。独立Logistic只用本折训练样本拟合方向概率；汇总后的样本外概率按时间前后切分，只有保序映射在更晚评估窗改善Brier分数才启用，之后才在全部样本外概率上重拟合生产校准器。程序同时保留近邻样本正收益比例和10%～90%经验区间作对照，并用每一时点之前的绝对残差计算滚动80%共形区间与实际覆盖率。这些统计量不是概率保证或置信承诺。

第一阶段内部可以完成模型工作以建立同一时点缓存，但公开结果不含预测点值、正收益比例和区间。第二阶段只取用户指定的一个 T+1、T+2 或 T+3 周期；只有该周期通过样本外验证且预测时点仍有效时才发布数值。

指定期限只有在以下条件全部满足时才可能标为“证据偏正面”：

1. 可交易性通过；
2. 对应期限完成并通过滚动样本外验证；
3. 给定测算资金满足对应板块的最低申报数量；
4. 扣费后预期达到配置门槛；
5. 独立方向模型的样本外概率（校准有效时使用校准值）达到配置门槛；
6. 技术面没有极端过热，基本面没有明显风险闸门。

否则按结果返回“证据中性”“证据偏负面”或“证据不足”。这些标签只归纳模型、成本、基本面、技术面和交易约束，不表示能买、应买或应卖。

### 5.6 未来三个交易日预测

第二阶段支持 `future_close` 模式。该预测以最近完整收盘日为 T，分别构造 T+1、T+2、T+3 三个未来市场交易日的收盘标签，目标是相对 T 日收盘的累计收益。股票在指定未来日期停牌或缺行情时，该样本为空，不会使用下一根日线顺延冒充。

三个周期分别使用同行面板、日频市场/估值/相对特征和三段扩展窗口滚动验证。训练样本的目标日期必须早于验证窗口，最后一段仍是必须通过的保留窗口。输出包括累计收益、按已知 T 日收盘换算并限制在合法涨跌幅区间内的参考收盘价、校准方向概率、80%共形/经验区间、质量标签和验证状态。参考收盘价只是模型结果的可读换算，不是目标价或收益承诺。

生成结果前还会核对当前上海时间与首个目标交易日：如果行情源未更新且首个目标日已经收盘，第二阶段返回时点失效并提示等待最新完整日线，禁止把已经发生的日期标成未来预测。

该预测与持有期情景分析相互独立：前者回答未来三个市场交易日的累计走势，后者用“假设下一交易日开盘”为成本测算基准并遵守 A 股 T+1 的日期约束。两者都只提供分析证据，不生成买卖指令，也不把日期包装成可执行计划。

## 6. 板块选股与预测

调用入口：`bankuai_xuangu`

### 6.1 板块与初筛

1. 根据名称匹配行业或概念板块，并返回最终识别的板块名、类型和相似度；相似度过低或候选接近时返回歧义错误。
2. 按名称排除 ST、退市标记和 N/C 上市初期特殊简称；当前不按上市天数过滤。
3. 应用价格上下限与成交额下限；缺少最新价格或成交额的成分股也会被排除。
4. 删除最新交易日接近涨停、次日难以成交的股票。
5. 按流动性限制历史行情拉取数量，仓库覆盖不足时最多研究 24 只；请求区间覆盖达标后最多 80 只。候选输出仍遵守单批 Top 8，训练池扩大不等于向用户一次推荐更多股票。

### 6.2 模型特征

每个交易日使用 19 个只依赖当日及过去数据的特征：

- 1/3/5/10/20 日动量；
- 价格相对 MA5/10/20/60 的距离；
- MA5 相对 MA20 的趋势；
- RSI、MACD；
- ATR、20 日波动率；
- 20 日回撤和区间位置；
- 量能比和单日振幅。

信号在 T 日收盘数据完整后形成，入口定义为下一市场交易日（T+1）开盘。输出 T+1 是入场后第 1 个可卖出交易日收盘，即信号后的第 2 个市场交易日；T+2、T+3 依次后移。目标是从实际入口开盘到相应退出收盘的累计收益，三个周期分别训练梯度提升树与Ridge集成。

板块流程只有一个外层时间留出窗口，因此不能直接用该窗口真实收益选择其自身权重。程序先在外层训练期内部再切一个尾部校准窗，并清除跨越内部切点的标签，用该嵌套窗选择外层验证权重；外层验证完成后，生产模型才可用完整外层样本外预测选择最终权重。

标签使用板块股票共同出现的市场日期定位。某只股票在规定的入口日或退出日停牌、缺行情时，该样本为空，不会把数日后的下一根日线误称为 T+1。

### 6.3 防止“回测看起来很好”

训练和验证按交易日期排序，不随机打乱。切分点之前的训练样本如果其目标日期跨过切分点，也会被删除，避免未来收益泄漏到训练集。

每个周期返回：

- 训练和验证样本数；
- MAE；
- 中位数基线 MAE；
- 相对基线提升；
- 涨跌方向准确率；
- 按日横截面 Rank IC；
- Top-N 扣成本平均收益、正收益日比例和相对板块股票的超额收益；
- 预测裁剪范围；
- `high` / `medium` / `low` 质量标签。

留出窗口只负责验证。验证指标计算完成后，生产预测模型会使用截至当前已经知道结果的全部标签重新训练，避免主动丢弃最近一段可用数据。

只有满足以下所有条件才进入 `validated_candidates`（验证后候选）：

1. 至少一个周期通过样本外验证；
2. 只有通过验证的周期参与加权和排名，剩余权重重新归一化；
3. 模型最强预测周期本身通过验证；
4. 给定测算资金足以覆盖所属板块的最低申报数量；
5. 加权成本后收益和最强周期成本后收益为正；
6. 用成本后收益除以持有期波动率/ATR 风险尺度得到的选择分数为正。

否则返回成功状态和 `no_validated_candidate_reason`，验证后候选数为 0。`model_ranking` 仅展示模型原始排序；候选与排序都不是买卖建议。

单批候选默认和上限都是 8。用户请求超过 8 只时，智能体先说明上限，再按第一批 Top 8 正常返回；没有 8 只通过证据门槛时不补足。结果包含由板块、完整收盘日、数据源、配置和有序候选代码生成的稳定 `selection_id`。后续“换一批”必须携带该编号与 `next_offset`，从第 9～16 名继续；如果行情快照或排序改变，旧序列作废并从新 Top 8 重启，防止跨时点混排或重复推荐。

### 6.4 当前局限

- 使用当前板块成分股和当前成交额靠前的最多 20 只股票回看历史，仍存在幸存者偏差与当前流动性选择偏差；
- 一次留出验证不等于跨多个牛熊周期稳定；
- 计划入场日一字涨停的样本会被判为无法成交并从收益标签中剔除；其他盘中排队、撤单和成交量约束仍无法由日线完整还原；
- 实际 T+1 开盘前不能可靠换算目标收盘价，因此 `predicted_close` 为空；模型未约束参考价和从信号收盘价推导的合法价格区间只用于解释边界，不能视为承诺成交价；
- 小样本、板块快速轮动、政策和突发事件会使模型失效；
- “预测到一个数字”不代表具有可交易优势，必须看样本外指标。

## 7. A 股规则与成本

- 当日买入最早下一个交易日卖出；本项目因此不输出入场当日收盘退出的不可执行收益；
- 自 2026 年 7 月 6 日起，沪深主板（含风险警示股票）按 10%、创业板/科创板按 20%、北交所按 30% 的通常涨跌幅规则归类；单股规则能识别 N/C 简称的无涨跌幅状态，板块选股会直接排除 N/C/退股票，重新上市或退市整理首日等少见特殊状态仍以交易所当日信息为准；
- 沪深主板和创业板买入通常按 100 股或其整数倍，科创板不少于 200 股且超过部分按 1 股递增，北交所不少于 100 股且超过部分按 1 股递增；
- 佣金通常买卖双向收取，并可能有每笔最低收费；
- 印花税只在卖出时收取；
- 滑点表示模型价格与实际成交价格之间的偏差。

印花税必须计入，因为 T+1 到 T+3 的预期收益本来就小。程序先按目标资金、个股参考入场价和申报数量规则计算实际可买股数，再计算费用；资金不足以满足最低买入数量时直接判定不可执行。

成本参数在根目录 [jiaoyi_chengben.json](./jiaoyi_chengben.json)，量化参数在 [lianghua_peizhi.json](./lianghua_peizhi.json)。税费和券商费率变化时，应修改配置后重新研究。

## 9. MCP 对外接口

MCP 客户端负责理解自然语言，本服务只负责取数和计算。支持 stdio 以及 HTTP `/mcp`；默认监听 `127.0.0.1`，但命令行允许显式修改 `--host`，对外监听时必须自行承担网络暴露风险。

MCP 永远只列出三个工具：

- `gupiao_fenxi`：第一阶段单股全面诊断，返回 `analysis_id`，不公开具体预测数值；
- `gupiao_yuce`：凭分析编号发布一个指定周期的验证后预测和可选持仓扣费收益；
- `bankuai_xuangu`：指定板块选股及 T+1/T+2/T+3 预测，每批最多 8 只并支持稳定顺延。

三个工具都不会改变券商或证券账户状态，也不提供破坏性操作。MCP 不提供 Shell、文件编辑、券商连接、账户读取、订单提交或自动交易能力；取数过程可能更新项目内的数据缓存。

## 12. 量化硬核程度与升级路线

### 12.1 当前客观定位

当前版本不是“几条均线让大模型拍脑袋”，但也还达不到机构级量化研究平台。更准确的评价是：**工程口径与防泄漏意识达到中等偏上，统计建模和数据基础设施处于个人研究项目的中等水平**。

已经具备的硬核部分：

- 预测变量全部来自严格日频数据：前复权个股日 K、上证/沪深 300/中证 1000 日 K、同行或板块日频宽度、相对收益，以及可取得时按同一交易日精确匹配的 `daily_basic` 估值、流通市值和换手因子；
- 全市场日频 SQLite 仓库按交易日持久化日线、复权、估值/规模/换手和交易日历，三端独立断点续传；完整覆盖时才扩大训练横截面；
- T+1、T+2、T+3 分别建模，不把一个技术评分冒充所有期限的预测；
- 标签按共同市场交易日定位，入口/退出停牌缺失不顺延，一字涨停入口样本剔除；
- 时序切分、跨界标签清除、三段扩展窗口、最终保留窗口、基线比较、Rank IC 和成本后 Top-N 指标共同约束；
- 每个训练窗口独立完成因子覆盖率检查、三段稳定性筛选、分位数去极值和流通市值残差化；验证段不参与筛选或预处理边界拟合；
- 收益回归使用梯度提升树/Ridge样本外加权集成，方向概率由独立正则Logistic模型产生；概率只在按时间后切的样本外窗口确实改善Brier分数时做保序校准，区间另做滚动共形覆盖检验；
- 佣金最低收费、过户费、卖出印花税、双边滑点、申报数量和资金可执行性进入结果；
- 预测发布有独立验证门禁，不再把“模型算出了数”误写成“这个数可信”。

仍显业余或不足的核心原因：

- 当前同行/板块成分和当前流动性回看历史，存在幸存者偏差、成分偏差和流动性选择偏差；
- 历史 `daily_basic` 依赖 Tushare 权限；接口不可用时相关因子会留空而不是向前填充。盈利质量等财务因子虽然按公告日用于诊断，但尚未进入训练；
- 默认约 3～4 年；无完整仓库时最多 20 只单股同行或 24 只板块股票，完整仓库可提高到 60/80，但仍无法充分覆盖多轮牛熊与所有风格切换；
- 已有独立方向模型，但板块最终排序仍基于经过成本和风险缩放的回归预测，尚未加入经独立验证证明有效的学习排序模型；
- 已记录确定性的实验指纹、折间稳定性和市场状态分层指标，但没有完整试验注册表、多重检验修正、Deflated Sharpe Ratio 或 PBO；
- 行业因子使用当前同行/当前板块的日频等权代理，不等同于历史时点行业指数成分。

因此当前模型适合做“有严格口径的短线研究原型”，不应包装成高频系统、机构 Alpha 平台或稳定盈利机器。

### 12.2 在现有日 K 数据条件下的优先升级

本轮日 K 路线的完成状态如下：

1. **PIT 数据约束（可得部分已完成）**：前复权行情只取完整收盘日；历史 `daily_basic` 只按股票代码和同一交易日精确合并；财务诊断只使用分析日之前已经公告的数据。当前数据源没有历史板块/行业成分快照，因此该部分的幸存者偏差仍明确保留，不能宣称完全消除。
2. **市场与行业日 K 因子（已完成）**：加入上证、沪深 300、中证 1000 日 K 收益和波动；加入同行/板块宽度、分散度、行业代理收益、市场参考收益和个股相对强弱。
3. **因子处理流水线（已完成）**：逐日流通市值中性化、横截面排名、训练窗覆盖率过滤、三段Rank IC符号稳定性筛选、训练窗去极值和Ridge/Logistic稳健缩放已经落地。
4. **目标专用模型（已完成当前范围）**：持有期收益和未来收盘累计收益分别训练；方向概率改由独立正则Logistic模型估计；板块排序继续使用成本后风险调整收益，暂不为复杂度而强加学习排序模型。
5. **小而可解释的模型集成（已完成）**：正则Ridge与梯度提升树只依据更早或嵌套样本外预测选择权重，并报告组件分歧。
6. **更严格验证（已完成当前可执行部分）**：单股三段扩展窗口和最终保留窗口、板块清除跨界标签的外层留出与三个时间稳定子窗、市场状态分层、折间离散度、实验SHA-256指纹均已加入。多重检验、DSR/PBO留作真正开展大规模策略试验后再做，当前不伪造统计结论。
7. **概率和区间校准（已完成）**：方向概率输出可靠性分桶；保序校准必须在更晚的独立时间窗改善Brier分数才启用；80%区间增加只使用先前残差的滚动共形覆盖检验，同时保留近邻经验区间作对照。
8. **扩大并缓存日 K 面板（已完成）**：常规单股同行20只、板块历史股票24只；新增全市场日频 SQLite 仓库，标准模式按交易日分别保存日线、复权因子和 `daily_basic` 并独立断点续传，低额度模式从 `daily.pre_close` 重建相对复权链。请求区间价格与复权覆盖达到90%后，训练上限自动提高到单股60只、板块80只；`daily_basic` 覆盖独立披露，缺失因子由训练窗筛选排除。复权日线仍保留12小时小缓存，指数日K继续增量缓存。
9. **历史日频估值/规模/换手因子（已完成可得部分）**：`pe_ttm`、`pb`、`circ_mv`、`turnover_rate` 已转为盈利收益率、账面市值比、流通市值对数、换手及横截面排名；没有同日值就留空。需要公告日的财务质量因子不混入本轮模型。

### 12.3 永久产品边界：只做日 K

本项目正式限定为**日 K 量化分析与未来三个交易日以内的预测**。30 分钟、60 分钟以及其他分钟级 K 线不属于延期事项，也不进入当前或后续既定路线图。市场、行业、估值、规模、换手、方向概率和区间校准等升级全部在严格日频口径内完成，不能用日 K 伪造分钟数据，也不能把盘中快照混入日 K 模型训练。

## 13. 当前已知限制

这些限制只描述现有 A 股研究结果的边界，不代表新增产品需求：

- 股票资料快照只从仓库首次抓取日开始积累；没有仓库建立前的历史时点板块/行业成分快照，因此无法完全消除幸存者偏差；
- 单股已使用三段滚动验证，板块外层留出也增加三个连续样本外稳定子窗；默认约 4 年历史仍不能覆盖所有牛熊、政策和风格周期；
- 单股同行池使用当前上市股票、当前行业标签和当前流动性回看历史，仍存在幸存者与当前成分偏差；
- 全市场仓库扩大的是训练横截面，不改变用户选股单批最多 8 只，也不代表仓库中的全部股票会在每次交互中同时入模；
- 单股盘中快照没有逐行交易日期，只能作可交易性参考，模型仍以完整收盘日为准；
- 基本面分数属于启发式初筛，没有完成所有行业的同行估值排名；
- 程序不连接券商、不读取账户、不下单，只提供分析和预测；用户自行作出最终判断。
